## 1. Одновимірні та двовимірні масиви

| Характеристика | Одновимірний масив | Двовимірний масив |
|----------------|-------------------|-------------------|
| Структура | Лінійна послідовність елементів | Таблиця «рядки × стовпці» |
| Доступ | `array[index]` | `matrix[row, col]` |
| Типове застосування | Фіксований набір однотипних даних | Сітка, матриця, карта координат |

### Приклади з проєкту

**Одновимірний масив** — оцінки лабораторних робіт студента:

```csharp
public byte[] LabGrades { get; } = new byte[10];

public void AddLabGrade(int labNumber, byte grade)
{
    LabGrades[labNumber - 1] = grade;  // доступ за індексом 0–9
}
```

**Двовимірний масив** — матриця портів 16×16:

```csharp
private readonly Port[,] _matrix = new Port[16, 16];

_matrix[row, col] = new Port { PortNumber = row * 16 + col + 1 };
```

Одновимірний масив `LabGrades` зберігає рівно 10 оцінок (фіксована кількість лабораторних). Двовимірний масив `Port[,]` моделює фізичну сітку портів, де кожна комірка — окремий об'єкт `Port`.

## 2. Ефективність StringBuilder

### Чому StringBuilder ефективніший?

При конкатенації рядків через `+` або `string.Concat()` кожна операція створює **новий об'єкт** у пам'яті, оскільки рядки в C# є **неізмінними (immutable)**. При тисячах операцій це призводить до надмірного навантаження на збірник сміття.

`StringBuilder` накопичує символи у **внутрішньому буфері**, змінюючи його без створення нових об'єктів на кожному кроці.

### Приклад з `PortLogger`

```csharp
private readonly StringBuilder _logBuilder = new();

public void LogOperation(string operation, int portNumber, string details)
{
    _logBuilder.Append('[')
        .Append(DateTime.Now.ToString("dd.MM.yyyy HH:mm:ss"))
        .Append("] Порт #")
        .Append(portNumber)
        .Append(" | ")
        .Append(operation.Trim())
        .Append(" | ")
        .Append(details.Trim())
        .AppendLine();
}
```

Кожна лабораторна робота, відкриття порту чи запис даних додає запис до **одного** буфера. Метод `GenerateFullReport()` у `StudentGroup` також використовує `StringBuilder` для формування багатосторінкового звіту без проміжних рядків.

**Порівняння:** 1000 операцій логування через `log = log + entry` створить ~1000 тимчасових рядків; `StringBuilder` працює з одним розширюваним буфером.

## 3. Симуляція портів вводу/виводу

Симуляція реалізована трьома класами:

| Клас | Роль |
|------|------|
| `Port` | Окремий порт із буфером 64 байти |
| `PortMatrix` | Сітка 16×16 портів |
| `PortLogger` | Журнал операцій |

### Використані масиви

1. **`byte[] DataBuffer`** (64 байти) у класі `Port` — буфер вводу/виводу, аналог апаратного регістра даних.
2. **`byte[] LabGrades`** (10 елементів) у класі `Student` — оцінки лабораторних.
3. **`Port[,] _matrix`** (16×16) у класі `PortMatrix` — двовимірна сітка портів.

### Цикл симуляції лабораторної роботи

```
Студент виконує лаб. → AddLabGrade() → byte[] LabGrades
                     → WriteToPort() → byte[] DataBuffer порту
                     → LogOperation() → StringBuilder логу
```

Метод `SimulateLabWork()` у `StudentGroup` об'єднує всі три кроки в одну бізнес-операцію.

## 4. Інтеграція масивів у систему управління студентами

Інтеграція виконана через клас `StudentGroup`, який координує студентів, матрицю портів і логер:

| Метод | Що робить |
|-------|-----------|
| `AssignStudentToPort()` | Прив'язує студента до комірки `Port[,]`, відкриває порт |
| `GetStudentsByPortStatus()` | Фільтрує студентів за станом їхнього порту |
| `SimulateLabWork()` | Записує оцінку в `LabGrades[]`, дані в `DataBuffer[]`, лог у `StringBuilder` |

**Словник прив'язок** `_studentPortAssignments` зберігає відповідність «номер залікової → координати (row, col)», що дозволяє знаходити порт студента без перебору всієї матриці при кожній операції.

Консольне меню (`Program.cs`) надає доступ до всіх операцій: від сортування лабораторних оцінок (п. 2) до симуляції лабораторної (п. 8).

## 5. Логіка роботи з матрицею 16×16

### Ініціалізація

```csharp
for (var row = 0; row < 16; row++)
    for (var col = 0; col < 16; col++)
        _matrix[row, col] = new Port { PortNumber = row * 16 + col + 1 };
```

Нумерація портів: **1–256** за формулою `row × 16 + col + 1`.

### Основні операції

| Метод | Опис |
|-------|------|
| `OpenPort(row, col)` | Встановлює `IsOpen = true` |
| `ClosePort(row, col)` | Встановлює `IsOpen = false` |
| `WriteToPort(row, col, data)` | Копіює до 64 байт у `DataBuffer` |
| `ReadFromPort(row, col)` | Повертає копію буфера |
| `ScanMatrix()` | Подвійний цикл, повертає всі відкриті порти |
| `FindOpenPortByNumber(n)` | Лінійний пошук у двовимірному масиві |

### Валідація координат

Усі методи перевіряють діапазон `0–15` для рядка та стовпця через `ValidateCoordinates()`.

### Візуалізація

`GetFormattedMatrixState()` формує таблицю 16×16 через `StringBuilder`, де `[O]` — відкритий порт, `[ ]` — закритий.

## 6. Труднощі при роботі з масивами

| Труднощі | Рішення |
|----------|---------|
| **Глибоке копіювання `byte[]` при `ICloneable`** | `Array.Copy()` для `DataBuffer` та `LabGrades` замість поверхневого `MemberwiseClone` |
| **Запис понад 64 байти в буфер порту** | `Math.Min(data.Length, 64)` обмежує розмір запису |
| **JSON-серіалізація двовимірного масиву** | Експорт у `List<PortCellSnapshot>` через `ExportState()` / `ImportState()` |
| **Прив'язка студент → порт** | Окремий `List<PortAssignmentEntry>` для збереження координат у JSON |
| **Сортування лабораторних оцінок** | `Array.Sort()` на копії масиву через `GetSortedLabGrades()`, оригінал не змінюється |
| **Пошук порту в матриці** | Подвійний цикл `for` по `row` і `col` з перевіркою `IsOpen` та `PortNumber` |

## 7. Масиви проти колекцій

| Критерій | Масив `T[]` | Колекція `List<T>` |
|----------|-------------|-------------------|
| Розмір | Фіксований | Динамічний |
| Швидкість доступу | O(1) за індексом | O(1) за індексом |
| Пам'ять | Компактний, без накладних витрат | Додаткові структури |
| Коли обирати | Відома кількість елементів | Кількість невизначена |

### Коли свідомо обрали масиви

| Масив | Чому не `List<T>` |
|-------|-------------------|
| `byte[10] LabGrades` | Рівно 10 лабораторних — фіксована навчальна програма |
| `byte[64] DataBuffer` | Апаратний буфер порту має фіксований розмір 64 байти |
| `Port[16,16] _matrix` | Сітка портів — класичний випадок двовимірного масиву |

### Коли залишили колекції

- `List<Student> _students` — кількість студентів у групі змінюється.
- `Dictionary<string, double>` у `GradeJournal` — предмети додаються динамічно.
- `List<Port>` у `ScanMatrix()` — результат пошуку невідомого розміру.

## 8. Що нового дізналися

1. **Практичне застосування `byte[]`** для моделювання низькорівневих буферів вводу/виводу.
2. **Двовимірні масиви `T[,]`** для представлення сіткових структур (матриця портів).
3. **`StringBuilder`** для ефективного накопичення логів і великих звітів.
4. **`ICloneable`** із глибоким копіюванням масивів — поверхнева копія недостатня.
5. **`Array.Sort()`** для сортування оцінок без LINQ-залежності в критичних місцях.
6. **Інтеграція різних структур даних** (масиви + словники + колекції) в єдину бізнес-логіку через `StudentGroup`.
