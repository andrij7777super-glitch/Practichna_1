## 1. Що таке перевантаження операторів? Які можна, а які ні?

Перевантаження операторів — це можливість задати власну поведінку для операторів (+, -, ==, > тощо) при роботі з об'єктами класів. Це робить код зрозумілішим, бо операції виглядають природно, як з простими типами.

Можна перевантажувати більшість операторів: арифметичні (+, -, *), порівняння (==, !=, <, >), унарні (++, --), true/false, а також явне і неявне приведення типів. Не можна перевантажувати оператори =, ., ?:, &&, ||, [] (окрім індексатора як окремої конструкції), new, typeof та деякі інші системні оператори.

## 2. Приклади 6 перевантажених операторів з коду

Перший приклад — оператор > для Student, який порівнює студентів за середнім балом і прогресом курсу. Другий — оператор + для Student, який об'єднує двох студентів у нового з усередненими балами. Третій — оператор + для StudentGroup, який зливає дві групи в одну. Четвертий — оператор + для Vector, який додає координати двох векторів. П'ятий — оператор true для GradePoint, який повертає true, якщо оцінка від 8 і вище. Шостий — явне приведення (double)Vector, яке повертає довжину вектора.

## 3. Різниця між явним (explicit) та неявним (implicit) приведенням

Неявне приведення виконується автоматично компілятором, коли це безпечно. У проєкті GradePoint неявно перетворюється на double і навпаки. Явне приведення потребує запису в дужках і використовується, коли можлива втрата даних або потрібна свідома дія програміста. Наприклад, (double)vector повертає довжину, а (double)examResult — відсоток успішності.

## 4. Для чого потрібні оператори true/false?

Вони дозволяють використовувати об'єкт у умовних виразах if і while. У GradePoint оператор true повертає true, якщо оцінка від 8 балів — це означає «добра оцінка». У ExamResult true означає, що студент склав іспит (не менше 60% балів). Замість написання if (result.Score / result.MaxScore >= 0.6) можна писати if (result).

## 5. Як реалізовано порівняння студентів?

Порівняння реалізовано через приватний метод Compare, який спочатку порівнює AverageGrade, а якщо бали рівні — CourseProgress. На його основі перевантажені оператори >, <, >=, <=, == і !=. Метод BestStudent() використовує оператор > для пошуку найкращого студента в групі.

## 6. Труднощі при виконанні та їх вирішення

Основна труднощ — оператор + для Student потребував валідного ПІБ з трьох слів і унікального номера залікової. Вирішено генерацією імені з прізвищ обох студентів і хеш-коду для номера залікової. Також при злитті груп виникала проблема дублювання студентів — додано перевірку перед AddStudent. Для порівняння Vector за довжиною реалізовано властивість Length.

## 7. Як це покращує читабельність

Замість group.MergeGroups(other) можна писати group + other. Замість FindStudent("24010001") — group["24010001"]. Порівняння student1 > student2 читається природніше, ніж виклик методу Compare. Умова if (gradePoint) зрозуміліша за перевірку gradePoint.Value >= 8.

## 8. Що нового дізналися

Навчився перевантажувати різні типи операторів, реалізувати індексатор для класу, використовувати true/false оператори для умовних перевірок, а також поєднувати явне і неявне приведення типів у власних класах Vector, GradePoint і ExamResult.
